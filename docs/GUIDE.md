# دليل moblies-shop — التوثيق الكامل

نظام **POS وإدارة مخزون** لمتجر موبايلات صغير. مبني على نفس تقنيات [StoreLedger](../StoreLedger) (Next.js + Prisma + Docker) لكن مخصص للبيع المباشر والمخزون.

---

## 1. ما تم بناؤه

### التقنيات

| الطبقة | التقنية |
|--------|---------|
| Frontend + Backend | Next.js 16 (App Router) — monolith |
| قاعدة البيانات | PostgreSQL 16 + Prisma 6 |
| المصادقة | Auth.js (Credentials + JWT + bcrypt) |
| الواجهة | Tailwind 4 + Radix UI (Shadcn-style) |
| الترجمة | next-intl — عربي (RTL) / إنجليزي |
| الفواتير | @react-pdf/renderer — PDF A4 |
| الاختبارات | Vitest |
| النشر | Docker Compose (standalone) |

### النموذج المعماري

- **متجر واحد** لكل deployment (ليس multi-tenant مثل StoreLedger)
- **ثلاثة أدوار:** Admin / Accountant / Sales (Employee قديم → يُعامل كـ Sales)
- الصلاحيات عبر `lib/permissions.ts` + Middleware
- **Clean Architecture**: منطق الأعمال في `lib/application/services/`
- **Server Actions** للعمليات (إضافة، تعديل، بيع، شراء...)
- **Prisma `$transaction`** لكل تغيير في المخزون

### قاعدة البيانات (Prisma)

|-------|---------|
| `User` | مستخدمون (ADMIN / ACCOUNTANT / SALES / EMPLOYEE) |

## 2. التثبيت والتشغيل

### Docker (موصى به للإنتاج)

```bash
cd moblies-shop
cp .env.example .env
```

عدّل `.env`:

```env
APP_PORT=3022
AUTH_URL=http://localhost:3022          # يجب أن يطابق عنوان الوصول
AUTH_SECRET=...                           # openssl rand -base64 32
ADMIN_EMAIL=admin@example.com
ADMIN_PASSWORD=كلمة-مرور-قوية
ADMIN_NAME=مدير المتجر
ADMIN_LOCALE=ar
```

```bash
docker compose up -d --build
```

افتح: **http://localhost:3022**

### تشغيل محلي (بدون Docker)

```bash
cp .env.example .env
# أضف DATABASE_URL=postgresql://postgres:postgres@localhost:5432/mobiles_shop

npm install
npx prisma migrate deploy
npm run db:bootstrap
npm run dev:clean -- --port 3002
```

افتح: **http://localhost:3002**

> **ملاحظة:** استخدم `npm run dev:kill` لإيقاف المنافذ 3001/3002 قبل إعادة التشغيل.  
> التطوير يستخدم `--webpack` لتجنب مشاكل cache في Turbopack.

### بيانات تجريبية (اختياري)

```bash
npm run docker:seed    # داخل Docker
# أو
npm run db:seed        # محلياً
```

> **ملاحظة:** الـ seed يحذف البيانات التشغيلية فقط — **لا يحذف** المستخدمين. يُنشئ:
> - **~47 منتج** (سامسونج، آيفون، شاومي، إكسسوارات، برمجة)
> - **~39 عميل** + ديون ودفعات
> - **~900+ عملية بيع** موزعة على **180 يوم**
> - **50 مشترى** + **36 مصروف** + مرتجعات + أوامر صيانة
> - موظف تجريبي: `employee@demo.shop` / `employee123`

### أوامر مفيدة

| الأمر | الوظيفة |
|-------|---------|
| `npm run dev` | تشغيل التطوير |
| `npm run dev:kill` | إيقاف منافذ 3001/3002 |
| `npm run dev:clean -- --port 3002` | تشغيل نظيف على منفذ محدد |
| `npm run build` | بناء الإنتاج |
| `npm run test` | تشغيل الاختبارات |
| `npm run docker:up` | تشغيل Docker |
| `npm run docker:down` | إيقاف Docker |
| `npm run docker:logs` | عرض logs |
| `npm run db:studio` | واجهة Prisma للبيانات |
| `docker compose down -v` | حذف كل البيانات والبدء من جديد |

## 3. دليل الاستخدام

### أول تشغيل

1. شغّل النظام (Docker أو محلي)
2. عند **قاعدة بيانات فارغة** يُنشأ تلقائياً:
   - مستخدم Admin من `.env`
   - مستودعان: Main Warehouse, Accessories Warehouse
   - فئات: Samsung, Apple, Xiaomi, Accessories, Programming
   - إعدادات المتجر الافتراضية
3. سجّل الدخول بالبريد وكلمة المرور من `.env`

### إعداد المتجر (Admin)

1. اذهب إلى **الإعدادات** (`/settings`)
2. عدّل:
   - **اسم المتجر** — يظهر في الشريط الجانبي والفاتورة (ليس من ملف الترجمة)
   - **الشعار** — رفع PNG/JPG
   - الهاتف والعنوان
   - بادئة الفاتورة (مثل `INV`)
   - **نص أسفل الفاتورة** — يظهر في PDF
   - **إظهار باركود نصي** — رقم الفاتورة بخط واسع
3. البادئة تُستخدم في أرقام الفواتير: `INV-2026-00001`

### إدارة المستخدمين (Admin)

1. **المستخدمون** (`/users`)
2. أضف مستخدمين بأدوار: **مدير** / **محاسب** / **مبيعات**
3. الأدوار موضحة في قسم الصلاحيات أدناه

### إعداد المخزون

#### المستودعات
- **المستودعات** → أضف (مثال: Main Warehouse, Accessories Warehouse)

#### الفئات
- **الفئات** → أضف فئة واربطها بمستودع (Samsung, Apple, Xiaomi...)

#### المنتجات
- **المنتجات** → أضف منتج:
  - الاسم، SKU (اختياري — يُولَّد تلقائياً إذا تُرك فارغاً)، Barcode (اختياري)
  - المستودع والفئة
  - **سعر التكلفة** و**سعر البيع** (التكلفة للمدير والمحاسب فقط)
  - الكمية وتنبيه انخفاض المخزون

#### نقل المخزون
- **نقل المخزون** → انقل كمية منتج من مستودع لآخر
- إذا المنتج غير موجود في المستودع الهدف → يُنشأ تلقائياً بنفس SKU

### المشتريات (زيادة المخزون)

1. **المشتريات** → نموذج شراء جديد
2. أدخل: اسم المورد، رقم فاتورة المورد، التاريخ
3. أضف المنتجات + الكمية + **سعر التكلفة**
4. عند الحفظ → **تزيد الكمية** تلقائياً

### نقطة البيع (POS)

1. **نقطة البيع** (`/pos`)
2. اختر **فئة** من التبويبات (Samsung, Apple, Accessories...) أو ابحث
3. اضغط على بطاقة المنتج → يُضاف للسلة
4. عدّل الكمية أو احذف من السلة
5. اختر **طريقة الدفع** بزر سريع:
   - **نقداً** (CASH)
   - **بطاقة** (CARD)
   - **آجل / دين** (ON_ACCOUNT) — **يجب** اختيار عميل
6. **إتمام البيع** → يُخصم المخزون + تُنشأ فاتورة
7. **طباعة** → PDF A4 عربي (بدون سعر التكلفة)

> **فئات POS** تأتي من أسماء فئات المنتجات في قاعدة البيانات (للمنتجات المتوفرة بمخزون > 0).

### قائمة المبيعات

1. **المبيعات** (`/sales`)
2. فلتر بالتاريخ: اليوم / هذا الأسبوع / نطاق مخصص
3. يعرض: رقم الفاتورة، العميل، البنود (اسم × كمية × سعر)، طريقة الدفع، الموظف
4. روابط: عرض الفاتورة | طباعة PDF

### الصيانة والبرمجة

| القسم | المسار | الوصف |
|-------|--------|-------|
| الصيانة | `/repairs` | طلب صيانة + تتبع الحالة |
| البرمجة | `/programming` | طلب برمجة + تتبع الحالة |

الحالات: جديد → قيد التنفيذ → جاهز → تم التسليم

> **ملاحظة:** أوامر البرمجة في `/programming` منفصلة عن بيع منتجات فئة Programming في POS.

### العملاء والديون

1. **العملاء** → أضف عميل (اسم، هاتف)
2. عند بيع **آجل** → يزيد `totalDebt` تلقائياً
3. من صفحة العميل → **تسجيل دفعة** لتقليل الدين
4. التقرير يعرض: إجمالي المشتريات، المدفوع، المتبقي

### المرتجعات

**مرتجع مبيعات:**
- يزيد المخزون
- يقلل الإيراد (في التقارير)
- إذا البيع كان آجل → يقلل دين العميل

**مرتجع مشتريات:**
- ينقص المخزون

### المصروفات

- **المصروفات** → أضف (إيجار، كهرباء، رواتب...)
- تُستخدم في حساب **صافي الربح**

### التقارير

| التبويب | المحتوى |
|---------|---------|
| مبيعات يومية | إجمالي اليوم + عدد العمليات + إجمالي الربح |
| مبيعات شهرية | إجمالي الشهر + تفصيل يومي |
| الأرباح | إجمالي الربح، صافي الربح، حسب المستودع، حسب الفئة |
| الديون | العملاء الذين عليهم ديون |
| المصروفات | قائمة المصروفات |
| مخزون منخفض | منتجات ≤ حد التنبيه |

**معادلات الربح:**
```
إجمالي الربح = Σ(سعر البيع − سعر التكلفة) × الكمية − مرتجعات
صافي الربح   = إجمالي الربح − المصروفات
```

## 4. قواعد المخزون (تلقائية)

| العملية | تأثير المخزون |
|---------|---------------|
| بيع (POS) | − كمية |
| شراء | + كمية |
| مرتجع مبيعات | + كمية |
| مرتجع مشتريات | − كمية |
| نقل مخزون | − من المصدر، + في الهدف |
| بيع آجل | + دين العميل |
| دفعة عميل | − دين العميل |

## 5. الصلاحيات

| الدور | Prisma | الصلاحيات |
|-------|--------|-----------|
| **مدير** | `ADMIN` | كل الصفحات + `/users` + `/settings` + نسخ احتياطي + تقارير الربح الكاملة |
| **محاسب** | `ACCOUNTANT` | + `/purchases` + `/expenses` + تعديل سعر التكلفة + تقارير مالية |
| **مبيعات** | `SALES` أو `EMPLOYEE` (قديم) | POS، مبيعات، مخزون (بدون تكلفة)، عملاء، صيانة، برمجة، تقارير أساسية |

**الملف:** `lib/permissions.ts` — `normalizeRole()`, `canManagePurchases()`, `canViewProfitReports()`, إلخ.

**Middleware:**
- `/users`, `/settings` → Admin فقط
- `/purchases`, `/expenses` → Admin + Accountant

## 6. استكشاف الأخطاء

| المشكلة | الحل |
|---------|------|
| Docker build فشل: `public not found` | تأكد وجود مجلد `public/` |
| لا أستطيع تسجيل الدخول | تحقق من `ADMIN_EMAIL/PASSWORD` وشغّل `db:bootstrap` |
| Auth error في الإنتاج | `AUTH_SECRET` و `AUTH_URL` يجب أن يطابقا عنوان الموقع |
| مخزون غير كافٍ عند البيع | راجع الكمية في **المنتجات** أو أضف **شراء** |
| Employee يرى Users | Middleware يمنعه — طبيعي |
| اسم المتجر مختلف في الشريط | تأكد من حفظ الإعدادات — الاسم يُقرأ من `ShopSettings` |
| فاتورة عربية مكسورة | تأكد وجود `public/fonts/NotoSansArabic-*.ttf` |
| خطأ بعد إتمام البيع | تم إصلاحه — `Button asChild` لا يمرر spinner |
| أرقام بـ `.00` | `formatMoney()` يعرض `200` بدلاً من `200.00` للأرقام الصحيحة |
