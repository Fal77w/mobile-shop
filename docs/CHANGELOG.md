# سجل التطوير — moblies-shop

**آخر تحديث:** 2026-06-22

هذا الملف يوثّق كل التحسينات والميزات المضافة خلال دورة التطوير الأخيرة.

## 1. بيانات تجريبية غنية (Seed)

| البند | العدد / المدة |
|--------|----------------|
| منتجات | ~47 |
| عملاء | ~39 |
| مبيعات | ~900+ |
| مشتريات | 50 |
| مصروفات | 36 (6 أشهر) |
| أوامر صيانة | ~159 |
| أوامر برمجة | (عبر `ProgrammingOrder` بعد التطوير) |
| حركات مخزون | ~2000+ |
| تاريخ البيانات | 180 يوم |

**الأمر:** `npm run db:seed` أو `docker compose --profile demo run --rm seed`

**ملاحظة:** الـ seed يحذف البيانات التشغيلية فقط (مبيعات، منتجات، عملاء...) ولا يحذف المستخدمين.

## 2. واجهة المستخدم (UI/UX)

### الخطوط والألوان
- خط **Tajawal** للعربية + **Inter** للإنجليزية (`app/layout.tsx`)
- لوحة ألوان SaaS (indigo primary) في `app/globals.css`
- مكونات UI محسّنة: Button, Card, Table, Dialog, Tabs, Badge

### التنقل
- شريط جانبي مجمّع: رئيسية، مخزون، عمليات، تقارير، إدارة
- شريط سفلي للموبايل: لوحة التحكم، نقطة البيع، المبيعات، العملاء
- **ثابت الارتفاع** للشريط الجانبي — أزرار الخروج والوضع الداكن دائماً ظاهرة
- أيقونات الخروج والوضع الداكن في الشريط العلوي أيضاً

### مكونات جديدة
- `DataTableShell`, `FormSection`, `EmptyState`, `LoadingSpinner`

## 3. نقطة البيع (POS) — إعادة تصميم

**المسار:** `/pos`

| قبل | بعد |
|-----|-----|
| بحث فقط + سلة فارغة | شبكة منتجات + فئات + بحث |
| قائمة منسدلة للدفع | 3 أزرار: نقداً / بطاقة / آجل |
| — | تبويبات فئات من قاعدة البيانات |

**مصدر الفئات:** أسماء فئات المنتجات (`Category.name`) للمنتجات المتوفرة بمخزون > 0.  
الترجمة العربية في `messages/ar.json` → `pos.categoryNames.*`

**تدفق البيع:**
1. اضغط المنتج → يُضاف للسلة
2. اختر طريقة الدفع
3. إتمام البيع → فاتورة + خصم مخزون

## 4. صفحة المبيعات

**المسار:** `/sales`

- عرض كل المبيعات مع فلتر **من / إلى**
- أزرار سريعة: **اليوم** | **هذا الأسبوع**
- بطاقات ملخص: عدد العمليات، الإجمالي، نقداً، بطاقة
- جدول يعرض: رقم الفاتورة، التاريخ، العميل، **المنتجات × الكمية × السعر**، طريقة الدفع، الموظف
- روابط: عرض الفاتورة | طباعة PDF

---

## 5. الصيانة والبرمجة

### الصيانة `/repairs`
- نموذج طلب صيانة جديد
- جدول أوامر مع تغيير الحالة: جديد → قيد التنفيذ → جاهز → تم التسليم
- نموذج Prisma: `RepairOrder`

### البرمجة `/programming`
- **مثل الصيانة** — نموذج + جدول + حالات
- نموذج Prisma: `ProgrammingOrder` (خدمة، جهاز، تكلفة...)
- منفصل عن بيع منتجات فئة Programming في POS

## 6. الصلاحيات — 3 أدوار

| الدور | Prisma enum | الصلاحيات |
|--------|-------------|-----------|
| **مدير** | `ADMIN` | كل شيء + مستخدمون + إعدادات + نسخ احتياطي |
| **محاسب** | `ACCOUNTANT` | + مشتريات + مصروفات + تقارير مالية |
| **مبيعات** | `SALES` أو `EMPLOYEE` (قديم) | POS، مبيعات، مخزون، عملاء، صيانة، برمجة |

**الملف:** `lib/permissions.ts` — `normalizeRole()`, `canManagePurchases()`, إلخ.

**Middleware:** `/users`, `/settings` → Admin فقط | `/purchases`, `/expenses` → Admin + Accountant

## 7. الإعدادات والعلامة التجارية

**المسار:** `/settings` (Admin)

| الحقل | الوظيفة |
|--------|---------|
| اسم المتجر | يظهر في الشريط الجانبي، الهاتف، الفاتورة |
| شعار المتجر | رفع PNG/JPG → `public/uploads/` |
| الهاتف / العنوان | على الفاتورة |
| بادئة الفاتورة | `INV-2026-00001` |
| نص أسفل الفاتورة | تذييل PDF |
| إظهار باركود نصي | رقم الفاتورة بخط واسع |

**مهم:** اسم المتجر في الواجهة يُقرأ من `ShopSettings` وليس من ملف الترجمة الثابت.

## 8. الفواتير PDF

**المسار:** `/api/invoices/[id]/pdf`

| الميزة | التفاصيل |
|--------|----------|
| حجم | A4 |
| خط عربي | Noto Sans Arabic (`public/fonts/`) |
| محتوى | اسم المحل، شعار، هاتف، عنوان، بنود، إجمالي |
| تذييل | من `invoiceFooter` |
| إصلاح | كان النص العربي مكسوراً (Helvetica لا يدعم العربية) |

**التسجيل:** `lib/pdf/register-fonts.ts`

---

## 9. تنسيق الأرقام

**الملف:** `lib/utils.ts` → `formatMoney()`

- بدون `.00` للأرقام الصحيحة: `200` بدلاً من `200.00`
- أرقام لاتينية (`1,234`) وليس هندية عربية
- الكسور تُعرض عند الحاجة فقط: `199.5`

---

## 10. SKU تلقائي

- عند إضافة منتج: **اترك حقل SKU فارغاً**
- يُولَّد رمز فريد تلقائياً: `SKU-XXXXXX`
- **الملف:** `lib/application/services/inventory-service.ts` → `generateUniqueSku()`


- موظف: `employee@demo.shop` / `employee123`
